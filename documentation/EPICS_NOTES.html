Introduction

This support module uses a LEEP driver to communicate with NASA ACQ nodes and provides an example IOC, and some Engineering CSS screens.  Most of the records mentioned in the following sections have names beginning with $(P)$(NODE) macros.  Typically the P macro is set to 'FDAS:' and the NODE macro to the acquisition node number (01 through 32) followed by a colon.

DRDY:status
 longin
 Most significant bit is set when DRDY signals from Quartz AD7768 ADCs are misaligned.
Least significant 20 bits are number of acquisition clock ticks (8 ns per tick) between event receiver PPS event and AD7768 ADC DRDY rising edge.  Values differing by more than 8 between nodes indicate that ADC synchronization between those nodes has been lost.
ADC:aligns
 longin
 Number of times that AD7768 ADC alignment has been requested.  ADC alignments are requested whenever the sampling frequency is set.
PPS:status
 longin
 The most significant bits contain information about the source of the hardware pulse-per-second signal:
Bit
 Description
27
 The node is acting as an event generator (NTP server address is non-zero).
28
 The primary PPS reference is valid.  For the event generator node this is the signal from the Quartz FMC board.  For all other nodes this is the event receiver PPS event.
29

30

31

The least significant 16 bits are estimate of the average jitter, in nanoseconds, in the pulse per second reference.

When acquisition is enabled acquisition activity is delayed until the middle of the next sample interval.  This ensures that there are no race conditions between nodes beginning to fill their acquisition buffers.  When acquisition is disabled acquisition activity continues until the currently-active packet has been filled and transmitted.

The time at which the firmware and software were built is contained in the per-node stringin records firmwareBuildDate and
softwareBuildDate, respectively.  The time at which the calibration factors were measured is contained in the per-node stringin record calibrationDate.  The status of the calibration file parsing and application is contained in the longin record calibrationStatus.  A value of 0 indicates that the contents of Calibration.csv are correct.  If that file is invalid the least significant 16 bits of calibrationStatus contain the line number where thing have gone wrong.  The most significant 16 bits contain an error code.  Some of the possible error codes are 3 (Calibration.csv does not exist), 5 (end-of-file was reached before all calibration values were read),  6 (syntax error), 7 (invalid character), and 8 (bad number).

The serial number of the Quartz FMC card is contained in the per-node longin record Quartz:serno.

The history of the Quartz analog to digital converter DRDY lines is captured in the longin record DRDY:history.  Each nibble, shown as a hexadecimal value, holds the status of the four DRDY lines with an 8 ns sampling clock.  The least significant bit of each nibble corresponds to the first ADC.  The least significant nibble is the value of the lines at the first rising edge of any of the lines after all lines have been low.  Sampling stops when all four lines are high.  The following table describes the conditions associated with some possible values where 'x' indicates a "don't care" value and 'y' indicates a value other then 'F':

The following table describes the records associated with the machine protection operations handled locally by each node.  The value oo is the machine protection mitigation output number, ranging from 01 to 08.  In addition to the records described below, the event generator node contains the bo record 'MPS:clear' that sends the event to clear the MPS 'tripped' latches in all the nodes when processed.


Name
 Type
 Description

 Bitmap specifying the channels for which an excursion below ACQ:LOLOlim:nn will trip the specified MPS output.
MPS:chkLO:oo longout
 Bitmap specifying the channels for which an excursion below ACQ:LOlim:nn will trip the specified MPS output.
MPS:chkHI:oo longout
 Bitmap specifying the channels for which an excursion above ACQ:HIlim:nn will trip the specified MPS output.
MPS:chkHIHI:oo longout
 Bitmap specifying the channels for which an excursion above ACQ:HIHIlim:nn will trip the specified MPS output.
MPS:chkInputs:oo longout
 Least significant eight bits are bitmap specifying the local MPS hardware inputs for which a 'bad' value will trip the specified MPS output.
MPS:goodState:oo longout
 Bitmap specifying the logic state (1/0) for which each local MPS hardware input will be considered 'good'.
MPS:status:oo longin
 Least significant bit is the local node 'tripped' status.  It is set when any of the conditions above are met and cleared by an event from the top-level node.  If any of the conditions above are met when this clear event arrives the 'tripped' status remains set and the 'first fault' values are updated.
The next-to-least significant bit is set when any of the conditions that would cause a trip are present.

Per-System Controls

These records are prefixed with only the $(P) macro.

Name
 Type
 Description
ACQ:enable
 bo
 Enable/disable (1/0) data acquisition.
ACQ:rate
 mbbo
 Set the sampling rate.
A set of per-node records monitor the rate and send it to their acquisition node.  Note that adding or changing a sampling rate requires changes to the FPGA code as well as to this record.
FPGA:reboot longout
 Writing 100 causes the acquisition nodes to reboot themselves.  A set of per-node longout records monitor this record and forward the value to their acquisition node. 

Internal Records

Records with names ending in an underscore are intended for internal use by the IOC and should not be written by any other clients.
